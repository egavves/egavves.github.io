System Architecture & Implementation

16.1 Feature API Layer

Every feature becomes a callable tool within the Calli Agent system. Tools are organized by computational complexity and latency.

Tool: analyze_face_count(video_url: str, timestamp_range: tuple) -> dict Returns: {face_count: int, face_positions: list, confidence: float} Tool: compute_emotional_valence_arc(video_url: str) -> dict Returns: {timeseries: [{t: float, valence: float}], peak_t: float, arc_type: str} Tool: extract_dialogue_phonemes(video_url: str, speaker_id: int) -> dict Returns: {phoneme_sequence: list, speech_rate: float, clarity_score: float} Tool: detect_cuts_and_transitions(video_url: str) -> dict Returns: {cut_times: list, cut_types: list, avg_shot_duration: float} Tool: measure_audience_reaction(event_id: str, moment_id: str) -> dict Returns: {sentiment_spike: float, social_velocity: int, tweet_rate: int} Tool: get_broadcast_competition(timeslot: str, market: str) -> dict Returns: {competing_shows: list, head_to_head_strength: float, market_share_prediction: float}

Tools are classified into four categories:

Instant Tools (<1 second)

Operate on static metadata: title, description, thumbnail, genre tag. No video processing required. Examples: format_type_classification, title_specificity_score, thumbnail_contrast_analysis, broadcast_timeslot_check.

Frame-Level Tools (10-30 seconds)

Require sampling video frames and running lightweight detectors. Examples: face_count, dominant_color_detection, shot_scale_distribution, text_overlay_density, scene_location_recognition.

Deep Tools (1-5 minutes)

Require full video processing with heavy ML models. Examples: emotional_valence_arc, dialogue_phoneme_sequence, music_intensity_profile, cut_frequency_analysis, camera_movement_classification, face_recognition_across_frames.

Live Signal Tools (Real-time)

Require API calls to external services: YouTube comments/engagement metrics, Reddit discussions, Twitter/X volume and sentiment, TikTok clip velocity, Twitch chat toxicity during live events. Computed in parallel during and immediately after broadcast.

16.2 Data Ingestion Daemons

Three daemon types manage continuous content discovery and processing:

Channel Watcher Daemon

Monitors a YouTube channel URL (or multiple channels). Triggers automatically when new video is detected. Auto-runs entire feature extraction pipeline on the new item. Pushes results to the Evidence Vault dashboard. Maintains historical comparison data per channel.

Topic Scout Daemon

Monitors YouTube search results for a keyword or phrase (e.g., "AI explained videos"). Discovers new content matching the profile. Runs feature extraction on each discovery. Flags anomalies relative to corpus baseline. Enables trend-following content strategy.

Live Event Daemon

Monitors a scheduled broadcast event (sports game, awards show, concert). Ingests real-time social signals during the event window (comments surge, sentiment shifts, viral clip detection). Records moment-by-moment engagement patterns. Post-event: computes full feature set + correlation with live engagement metrics.

All daemons feed into a unified processing queue:

[Event Trigger] → [Content Ingester] → [Feature Extraction Queue] → [ML Pipeline] → [Dashboard Push] → [Alert Generation]

16.3 ML & Causal Pipeline

The core intelligence stack consists of five layers:

Layer 1: Correlation Engine

Computes Pearson and Spearman correlations across all 320+ features vs outcome metrics (views, watch-time, CPM, engagement rate, completion rate). Identifies simple linear relationships. Example: face_count r=0.33 with views on YouTube. Fast baseline for understanding feature importance.

Layer 2: Causal Discovery

Uses Peter-Clark (PC) Algorithm to construct a causal graph (DAG) from observational data. Identifies which features causally precede performance outcomes vs are mere correlates or confounded. Critical insight: correlation ≠ causation. This layer separates spurious correlations from true drivers.

Layer 3: Predictive Models

Trains ensemble models (Support Vector Regression, Gradient Boosted Trees, Multi-Layer Perceptron neural networks) on extracted features, segmented per content category. Predicts expected performance range (views, engagement, virality) before content release. Enables A/B testing of creative changes.

Layer 4: Causal Intervention Simulation

Given the causal DAG from Layer 2, simulates counterfactual scenarios: "If I increase this feature (e.g., emotional valence) while holding confounders constant, what happens to expected viewership?" This is the product's core recommendation engine. Translates causality into actionable creative guidance.

Layer 5: LLM Synthesis

Claude Agent synthesizes all quantitative signals (correlations, causal estimates, model predictions) into natural language hypotheses. Every claim is explicitly tagged with uncertainty bounds and data source citations. Output: Creative Brief with ranked recommendations and confidence levels.

16.4 Dual Dashboard Architecture

Two complementary interfaces serve different user needs:

Evidence Vault (Full)

Comprehensive quantitative reference for technical stakeholders.

face_count: 2.3 ± 0.4 (YouTube content)
Correlation with Views: r = 0.33, p < 0.05
Causal Status: Confounded by content_length (PC test)
Predicted View Range: 250K - 1.2M (95% CI)

Creative Brief (Compressed)

Executive summary with actionable recommendations.

⚡ Key Driver 1: Emotional arc peaks at 65% through video (strong completion signal)
⚡ Hypothesis: Increasing cold-open hook quality could add 15-20% to early retention
⚡ Recommendation: Strengthen second-act emotional payoff (currently flat)
🔗 Confidence: Medium (supported by 3+ causal paths)

Evidence Tagging System

Every claim in both dashboards is tagged to indicate its source and confidence level:

[QUANTITATIVE] Raw feature value with confidence interval. Underlined in blue. Links to raw data when clicked.
[AGENT INFERENCE] Claude Agent synthesis of multiple signals. Highlighted in gold. Clearly labeled as LLM output, not ground truth.
[CAUSAL CLAIM] Backed by PC Algorithm or RCT data. Orange border. Indicates direction of effect, not just correlation.
[CORRELATION ONLY] Correlation without causal test. Grey tag. Useful for hypothesis generation but not for intervention planning.
